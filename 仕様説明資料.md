## 1. システムの概要

**法務クイズ研修**は、法務・コンプライアンスの知識をRPG（ロールプレイングゲーム）形式で学ぶ、ブラウザで動く社内研修ツールです。

### ポイント

- **役職ステージ制**：「新入社員」→「CLO（最高法務責任者）」まで、6つのステージを順番に進む
- **コース選択**：総合（横断）に加え、法分野に特化した専門コースを選んで受講できる
- **4択クイズ**：会社法・個人情報保護法・独禁法・労働法など、法務・コンプラの幅広いテーマを出題
- **ゲーム感覚で学習**：タイマー・コンボ・コンディション（HP）などのゲーム要素で飽きずに続けられる
- **インストール不要**：ブラウザだけで動く（HTMLファイルを開くだけ）

## 2. 起動方法

### 通常の手順

1. `index.html` をブラウザで開く
2. コースを選択（例：総合コース / 個人情報保護法コース）
3. 受講者名を入力して「▶ 開始」をクリック

> **注意**：`questions.json`（問題データ）を読み込む関係上、**ファイルを直接ダブルクリックでは動かない場合があります**。
> その場合は以下の方法で起動してください。
>
> ```
> npx serve .
> ```
> （Node.js が入っていれば上記コマンドをフォルダ内で実行 → ブラウザで `http://localhost:3000` を開く）
>
> または VS Code の「Live Server」拡張機能を使用。

```

## 4. 各ステージの内容

各コースは全6ステージで構成され、各ステージで**5問**出題されます。

> 注意：ステージの「出題テーマの目安」は **コースによって異なります**。下表は「総合コース」の目安です。

| ステージ | 役職名 | 出題テーマの目安（総合コース） |
|---------|--------|--------------|
| Stage 1 | 新入社員 | 法律用語の整理、会社法・民法・独禁法の入口、労働法・個人情報の基本（入門） |
| Stage 2 | 一般社員 | 個人情報保護法、職安法、会計・財務・人事と法リスクの接続（基礎） |
| Stage 3 | 係長 | 契約書審査、会社法・民法・独禁法・不競法・金商法の入口、GDPR（中級） |
| Stage 4 | 課長 | 契約・人事・財務・戦略など複数論点の横断判断（中級〜実務） |
| Stage 5 | 部長 | CSRD・サステナ開示、EU AI法、GDPR、上級ガバナンス（上級） |
| Stage 6 | CLO | EU AI法・金商法・経営横断の総合力（最上級） |

各問題は **`course`（任意）** と **`stage`（1〜6）** で出題枠が決まります。
- `course` 未指定の問題は **総合コース（general）** 扱い
- `course` 指定の問題は **指定された専門コース** でのみ出題（例：`ppc`）

各ステージは**同一コース内で前のステージをクリアしないと解放されません**（ステージマップで確認可能）。

## 5. クイズの仕組み（ルール詳細）

### 5-1. 出題のルール

- **1ステージあたり5問**出題
- ステージにひもづく問題プールからランダムに5問を選出
- 選択肢の順番も毎回シャッフル（正解の位置が変わる）
- 問題文には **テーマ名** が表示される（例：`[会社法]`）

### 5-2. タイマー

- 1問あたり **30秒**
- タイマーバーで残り時間を視覚的に確認可能
  - 残り20秒以上：緑
  - 残り10〜20秒：黄
  - 残り10秒以下：赤（警告色）
- **時間切れ = 不正解扱い**（コンディション -20）

### 5-3. 正解・不正解の判定

| 結果 | 通常問題 | 最終問題（ステージ最後の1問） |
|------|---------|--------------------------|
| 正解 | 習熟pt 獲得（ステージ基礎値×コンボ等） | 習熟pt 獲得・ステージクリア |
| 不正解 | コンディション -20 | **コンディションが0になる**（ステージ終了） |
| 時間切れ | コンディション -20 | **コンディションが0になる**（ステージ終了） |

> **最終問題**はステージの5問目です。画面上部に警告が表示され、効果音も変わります。ここで誤答・時間切れになると、そのステージは強制終了となりセッションが終わります。

### 5-4. コンボボーナス

連続正解すると習熟ptの倍率が上がります。

| 連続正解数 | 倍率 |
|-----------|------|
| 1問 | ×1.0（通常） |
| 2問 | ×1.5 |
| 3問 | ×2.0 |
| 4問以上 | ×2.5 |

不正解・時間切れでコンボはリセットされます。

### 5-5. 速答ボーナス

素早く回答すると追加で習熟ptが加算されます。

| 残り時間 | ボーナス倍率 |
|---------|------------|
| 25秒以上 | +50%分 |
| 20〜24秒 | +30%分 |
| 15〜19秒 | +10%分 |
| 14秒以下 | なし |

## 6. ステータス・数値の説明

クイズ中は常に以下のステータスが表示されます。

| 表示名 | 説明 |
|--------|------|
| **名前** | 入力した受講者名 |
| **役職** | 現在のステージの役職名（Stage 1 = 新入社員 など） |
| **段階** | レベル。習熟ptが `段階 × 50` を超えると1上がる |
| **コンディション** | いわゆるHP。最初は100。不正解・時間切れで -20。0になるとセッション終了 |
| **習熟pt** | 正解で獲得するポイント。コンボ・速答で加算される |
| **進行** | 現ステージで何問目か（例：2/5） |
| **ステージ** | 現在のステージ番号と名前 |

## 7. 評価・スコアの計算方法

### ステージごとの評価（S / A / B / C）

各ステージ終了時に評価が出ます。計算式は以下のとおりです。

```
総合スコア = (正答率 × 0.7) + (コンディション維持率 × 0.3)

  S：0.90以上　「正答率とコンディションがともに高水準」
  A：0.75以上　「高い理解度」
  B：0.55以上　「達成基準を満たす水準」
  C：0.55未満　「復習を推奨」
```

### 最終スコア（セッション終了・全クリア時）

```
最終スコア = 習熟pt × 10 + コンディション × 5 + 段階 × 50 + 到達ステージ数 × 200
```

正解時の習熟ptは、**ステージごとの基礎値**にコンボ倍率・速答ボーナス（ステージ基礎値に対する割合）を加味して算出されます。

最終スコアは**ランキングに自動登録**されます（上位10件を保存）。

## 8. 便利機能一覧

### 8-1. 途中保存・再開

- セッション中の状態は**自動的にブラウザに保存**されます
- 次回アクセス時に「保存データがあります。続きから再開しますか？」と確認が出る
- 「いいえ」を選ぶと保存データが削除されて最初から

### 8-2. 復習モード

- 不正解・時間切れになった問題が**復習リスト**に自動登録される
- トップ画面の「📖 復習 ○問」ボタンから、苦手問題だけを集中的に練習できる
- 復習モードで正解すると、その問題は復習リストから外れる
- **復習モード中はコンディション・習熟ptの増減なし**（純粋な練習用）

### 8-3. ステージマップ

「📍 ステージマップ」ボタンで、各ステージのクリア状況を一覧表示。

| 表示 | 意味 |
|------|------|
| ✅ 修了済 | クリア完了 |
| ▶ 受講可能 | 次に進めるステージ |
| 🔒 | まだ解放されていない |

### 8-4. 正答率統計

「📊 正答率統計」ボタンで、テーマ別の正答率をバー表示。

| バーの色 | 正答率 |
|---------|--------|
| 緑 | 80%以上 |
| 黄 | 60〜79% |
| 赤 | 59%以下 |

### 8-5. ランキング

「🏆 ランキング」ボタンで過去のスコア上位5件を表示。
セッション終了・全ステージ修了時に自動登録されます。

### 8-6. 結果コピー

セッション終了・全クリア時に「📋 結果をコピー」ボタンが表示され、結果テキストをクリップボードにコピーできます。

## 9. 問題データの管理

### 問題ファイルの場所

問題はすべて `questions.json` というファイルに格納されています。

### 問題の項目

各問題は以下の情報で構成されています。

| 項目名 | 内容 | 例 |
|--------|------|----|
| id | 問題の識別番号 | `q001` |
| theme | テーマ・法分野 | `会社法` |
| course | 所属コースID（任意）。未指定の場合は総合コース扱い | `ppc` |
| stage | 出題ステージ（1〜6）。この番号のプールから出題 | `3` |
| tags | 関連キーワード | `["株主総会", "取締役会"]` |
| text | 問題文 | `取締役会設置会社で…` |
| choices | 4つの選択肢 | `["代表取締役", "監査役", …]` |
| answer | 正解の番号（0〜3） | `2`（3番目の選択肢が正解） |
| explanation | 解説文 | `取締役会設置会社では…` |

### 問題の編集方法（admin.html）

`admin.html` をブラウザで開くと、問題を**フォーム形式で編集**できます。

#### 手順

1. `admin.html` をブラウザで開く
2. `questions.json` をドラッグ&ドロップ（またはクリックして選択）
3. 左のリストから編集したい問題をクリック
4. 右側のフォームで内容を編集
   - 問題文・選択肢・正答（ラジオボタン）・解説
   - テーマ・タグ・ステージ
5. 「この問題の変更を確定」ボタンで編集内容を確定
6. 「💾 JSONをダウンロード」ボタンを押す
7. ダウンロードした `questions.json` を元のファイルに上書き保存

#### admin.htmlの機能

| 機能 | 説明 |
|------|------|
| 検索 | 問題文・テーマ・IDで絞り込み |
| フィルター | ステージ・テーマで絞り込み |
| 変更マーク | 編集済みの問題に赤丸が表示される |
| 問題追加 | 右上「＋ 問題追加」から新規作成 |
| 問題削除 | 編集パネル右上「削除」ボタン |

> **保存のタイミング**：`admin.html` 上での変更は「JSONをダウンロード」するまで本番ファイルには反映されません。必ず上書き保存まで行ってください。

## 10. ファイル構成

```
legal-rpg/
│
├── index.html          ← 研修本体（ブラウザで開くファイル）
├── app.js              ← ゲームのロジック全体
├── style.css           ← 画面デザイン
├── bg.js               ← 背景エフェクト
├── sound.js            ← 効果音・BGM
│
├── questions.json      ← 問題データ（ここを編集して問題を管理）
│
├── admin.html          ← 問題編集ツール（問題の追加・修正に使用）
│
└── README.md           ← 簡易起動手順
```

### 各ファイルの役割まとめ

| ファイル | 触る頻度 | 用途 |
|----------|---------|------|
| `questions.json` | よく触る | 問題・解説の追加・修正 |
| `admin.html` | よく触る | questions.json を編集するためのツール |
| `index.html` | ほとんど触らない | 研修本体の画面構造 |
| `app.js` | ほとんど触らない | ゲームのルール・ロジック |
| `style.css` | 必要に応じて | 見た目のデザイン調整 |

## よくある質問

**Q. 保存データを消したい**
A. ブラウザの「再起動」ボタンを押すと保存データが削除されます。または、ブラウザの開発者ツール → Application → LocalStorage から `legal_rpg_save_v3` を削除。

**Q. 問題を増やしたい**
A. `admin.html` で追加後、`questions.json` を上書き保存してください。

**Q. 特定のステージだけやり直したい**
A. 現在はステージ選択機能はなく、Stage 1 から順番に進む仕様です。

**Q. 研修結果を記録・集計したい**
A. 現在はブラウザのローカルストレージに保存される仕組みです。他の端末には引き継がれません。
